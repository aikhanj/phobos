Design Section 1: Procedural Texture System

New module apps/web/src/game/textures.ts — canvas-drawn textures for every surface type

How It Works

A single textures.ts module exports a getTexture(type) function that returns cached THREE.CanvasTexture instances. Each texture is drawn procedurally on an offscreen canvas at PS1-authentic resolution (64×64 or 128×128), with NearestFilter and RepeatWrapping so they tile and stay pixelated.

Scenes call getTexture('wood_dark') instead of passing a hex color. The makeWall and makeProp helpers get updated to accept an optional texture parameter.

Texture Types Needed

Walls
  • concrete — mottled grey, subtle cracks (basement)
  • concrete_dark — darker variant, damp stains
  • wallpaper — faded stripe pattern (bedroom)
  • wallpaper_torn — same but with ripped sections
  • wood_panel — vertical plank lines (attic)
Floors
  • concrete_floor — cracked, dirty (basement)
  • wood_floor — plank lines, worn grain (bedroom/attic)
Props
  • wood_dark — dark grain (crates, shelves, furniture)
  • wood_light — lighter variant (bed frame, chair)
  • metal — brushed, tarnished (pipes, lantern)
  • fabric — cross-hatch weave (curtains, rug, sheets)
  • plaster — rough, crumbly (ceiling)
Special
  • door — paneled wood with frame lines
  • brick — exposed brick (basement accent)
  • tile — cracked ceramic (if bathroom area)

Integration With Existing Code

Before:

this.makeProp(0.6, 0.6, 0.6, pos, 0x2a1f14)  // solid brown box

After:

this.makeProp(0.6, 0.6, 0.6, pos, 'wood_dark')  // textured crate

makeProp and makeWall accept either a hex color (backwards compatible) or a texture name string. When given a string, they call getTexture() internally.
